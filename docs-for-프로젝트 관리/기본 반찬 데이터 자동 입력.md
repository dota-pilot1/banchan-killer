# 기본 반찬 데이터 자동 입력 프로세스

이 문서는 서버 실행 시 기본 반찬 데이터가 자동으로 입력되는 흐름을 정리한 문서입니다.

## 대상 코드

- [`ProductDataInitializer.java`](/Users/terecal/banchan-killer-container/banchan-killer-api/src/main/java/com/banchan/product/infrastructure/ProductDataInitializer.java)

## Step 1. Spring Boot 서버가 시작된다

- Spring Boot 애플리케이션이 실행되면 `CommandLineRunner`를 구현한 `ProductDataInitializer`가 함께 실행됩니다.
- 이 클래스는 애플리케이션 시작 직후 한 번 호출되어 기본 상품 데이터를 점검합니다.

## Step 2. 코드 안에서 기본 샘플 상품 목록을 만든다

- `run()` 메서드 안에서 `sampleProducts` 리스트를 먼저 생성합니다.
- 현재 기본 반찬 데이터는 아래와 같습니다.

1. `꽈리고추 볶음`
2. `콩나물 무침`
3. `메추리알 볶음`
4. `돼지불고기`
5. `파김치`
6. `배추김치`

- 각 상품에는 `name`, `description`, `price`, `stockQuantity`, `category`, `imageUrl` 값이 함께 들어갑니다.

## Step 3. 현재 DB에 저장된 상품 개수를 확인한다

- `productJpaRepository.count()`를 호출해서 현재 `products` 테이블에 상품이 이미 들어 있는지 확인합니다.
- 여기서는 상품이 `0개인지`, 즉 완전히 비어 있는지만 확인합니다.

## Step 4. 상품이 이미 있으면 아무 것도 하지 않고 종료한다

- `productJpaRepository.count() > 0`이면 기존 상품 데이터가 있다는 뜻입니다.
- 이 경우 `return`으로 바로 종료하고, 기존 상품 데이터는 그대로 유지합니다.
- 따라서 관리자 페이지에서 수정한 가격, 설명, 이미지 같은 값이 서버 재시작으로 초기화되지 않습니다.

## Step 5. 상품이 0개일 때만 기본 반찬 데이터를 저장한다

- DB가 비어 있을 때만 `productJpaRepository.saveAll(sampleProducts)`를 호출합니다.
- 이 단계에서 기본 반찬 6종이 최초 1회 입력됩니다.

## Step 6. 이후 재시작 시에는 기존 상품 데이터를 유지한다

- 한 번 데이터가 들어간 뒤에는 서버를 다시 실행해도 자동으로 덮어쓰지 않습니다.
- 따라서 관리자 페이지에서 수정한 상품 정보나 새로 등록한 상품이 그대로 유지됩니다.

## 정리

- 서버 시작 시 `ProductDataInitializer`가 실행됩니다.
- 상품이 `0개`일 때만 기본 반찬 6종을 입력합니다.
- 기존 상품 데이터가 있으면 아무 것도 하지 않습니다.
- 이 로직은 빈 개발 DB를 빠르게 채우기 위한 초기 데이터 입력 방식입니다.
