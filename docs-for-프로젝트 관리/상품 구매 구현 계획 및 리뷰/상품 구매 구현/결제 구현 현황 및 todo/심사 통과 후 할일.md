## 심사 통과 후 체크리스트

### 1. 실결제 키 교체

심사 통과 시 **라이브 키**가 활성화됨.

```
# 프론트 (.env)
VITE_TOSS_CLIENT_KEY=live_gck_xxxxxxxxxxxxx    # 결제위젯 클라이언트 키

# 백엔드 (.env)
TOSS_CLIENT_KEY=live_ck_xxxxxxxxxxxxx
TOSS_SECRET_KEY=live_sk_xxxxxxxxxxxxx
```

> 테스트 키 → 라이브 키로 교체만 하면 됨. 코드 변경 없음.

### 2. 웹훅 URL 등록

토스 대시보드 → 개발 정보 → 웹훅 URL 설정:

```
https://your-domain.com/api/payments/webhook
```

> 현재는 웹훅 미구현. 필요 시 추가 (가상계좌 입금 확인 등)

### 3. 실결제 테스트

- [ ] 실제 카드로 100원 결제 테스트
- [ ] 결제 완료 후 주문 상태 PAID 확인
- [ ] 토스 대시보드에서 거래 내역 확인
- [ ] 100원 결제 취소 처리

### 4. 홈페이지 심사 준비

토스 심사 시 홈페이지를 확인함. 필요 사항:
- 홈페이지 하단에 상호명 표시
- 판매 가능한 상품 1개 이상 등록
- 빈 카테고리 없어야 함
- 상품 이미지 정상 표시

### 5. 실결제 전환 후 추가 구현 (선택)

- [ ] 결제 취소 API (`POST /api/payments/{paymentKey}/cancel`)
- [ ] 환불 처리 로직
- [ ] 웹훅 수신 엔드포인트 (`POST /api/payments/webhook`)
- [ ] 관리자 대시보드 결제 내역 조회
