# 주문서 작성 구현 계획

## 현재 진행 상태

**1차~3차 구현 완료. 4차(실결제 전환) 대기 중.**

---

## 단계별 구현 현황

### 1차 구현 ✅ 완료

주문서를 쓸 수 있게 만드는 단계

- [x] 장바구니 선택 체크박스
- [x] 선택 상품만 주문 흐름으로 넘기기
- [x] 주문서 페이지 진입
- [x] 주문 상품/금액 요약 표시

### 2차 구현 ✅ 완료

주문서가 실제 사용자 데이터와 연결되는 단계

- [x] 프로필 API 연동
- [x] 배송지 API 연동 (다중 배송지 + 기본 배송지)
- [x] 주문서에서 기본 배송지 자동 채움
- [x] 주문 생성 API 연동
- [x] 주문 목록 / 주문 상세 페이지
- [x] 전화번호 자동 포맷팅 + 유효성 검사

### 3차 구현 ✅ 완료

토스페이먼츠 결제 연동 + 장바구니 DB 마이그레이션

- [x] 토스페이먼츠 SDK 연동 (API 개별 연동 방식)
- [x] 결제 승인 API (`POST /api/payments/confirm`)
- [x] 금액 위변조 검증 + 중복 결제 방지
- [x] payments 테이블 저장 (결제수단, 승인시각, 영수증 URL)
- [x] 주문 상태 `PENDING_PAYMENT → PAID` 자동 변경
- [x] 결제 성공/실패 페이지
- [x] 결제 내역 목록 + 결제 상세 페이지
- [x] 장바구니 localStorage → DB 마이그레이션
- [x] 결제 완료 시 장바구니에서 결제 상품 자동 삭제
- [x] 주문 상세에서 결제 상태별 UI 분기

---

## 이후 구현 계획

### 4차 구현 — 실결제 전환

토스페이먼츠 심사 통과 후 실결제 활성화

- [ ] 테스트 키 → 라이브 키 교체 (코드 변경 없음)
- [ ] 결제위젯 연동 키 발급 후 위젯 방식으로 전환
- [ ] 실제 카드로 소액 결제 테스트
- [ ] 토스 대시보드 거래 내역 확인

### 5차 구현 — 운영 기능

- [ ] 재고 차감 (결제 시 차감, 취소 시 복구, 동시성 제어)
- [ ] 주문 취소 + 환불 (토스 취소 API 연동)
- [ ] 배송 상태 관리 (SHIPPED, DELIVERED)
- [ ] 관리자 주문 처리 대시보드

### 6차 구현 — 고도화

- [ ] 쿠폰 / 포인트 / 할인 정책
- [ ] 주소 검색 API 연동 (카카오/네이버)
- [ ] 비회원 주문
- [ ] 리뷰/평점
- [ ] 정기배송/재주문

---

## 한 줄 정리

3차까지 완료 = 장바구니 → 주문 → 결제 → 결제 내역 전체 플로우 동작.
4차(실결제 전환)는 토스 심사 통과만 기다리면 키 교체로 즉시 가능.
