# 주문서 작성 3차 구현 계획

## 3차 목표

3차 구현의 목표는
`생성된 주문 -> 토스페이먼츠 결제 -> 주문 상태 확정`
까지 마무리하는 것이다.

즉 2차에서 만든 주문 데이터를
실제 결제 가능한 주문으로 완성하는 단계다.

---

## PG사 선택: 토스페이먼츠 (Toss Payments)

### 선택 이유

- React 공식 SDK 제공 (`@tosspayments/tosspayments-sdk`)
- 연동 코드가 심플 (프론트 10줄 내외)
- 카드/계좌이체/카카오페이/네이버페이/토스페이 한 번에 지원
- 한국어 문서 + 예제 풍부
- 테스트 모드 무료 (실결제 없이 전체 흐름 검증 가능)
- Spring Boot 연동 가이드 공식 제공

### 참고 문서

- https://docs.tosspayments.com/guides/v2/get-started
- https://docs.tosspayments.com/reference/widget-sdk

---

## 3차 범위

### 포함

- 토스페이먼츠 결제 위젯 SDK 연동
- 결제 요청 → 승인 → 확정 전체 흐름
- `payments` 테이블 저장
- 주문 상태 `PENDING_PAYMENT -> PAID` 변경
- 결제 성공 / 실패 화면
- 금액 위변조 검증 (백엔드)

### 제외

- 부분 취소 / 부분 환불
- 포인트 / 쿠폰 / 할인 정책
- 관리자 정산 / 매출 통계
- 비회원 주문
- 가상계좌 입금 대기 (1차 결제 연동에서는 즉시 결제만)

## 프론트엔드 작업 항목

### 0. 사전 완료

- [완료] 결제 페이지 플레이스홀더 생성 (`/payment/:orderId`)
- [완료] 주문 생성 후 결제 페이지로 자동 이동

### 1. 토스페이먼츠 SDK 설치

```bash
npm install @tosspayments/tosspayments-sdk
```

### 2. 결제 페이지 구현 (`PaymentPage.tsx`)

현재 플레이스홀더를 실제 결제 위젯으로 교체:

```typescript
import { loadTossPayments } from '@tosspayments/tosspayments-sdk';

// 위젯 초기화
const tossPayments = await loadTossPayments(CLIENT_KEY);
const widgets = tossPayments.widgets({ customerKey: userId });

// 금액 설정
await widgets.setAmount({ currency: 'KRW', value: order.totalAmount });

// 결제 수단 위젯 렌더링
await widgets.renderPaymentMethods({ selector: '#payment-method' });

// 결제 요청
await widgets.requestPayment({
  orderId: order.orderNumber,
  orderName: '꽈리고추 볶음 외 1건',
  successUrl: `${window.location.origin}/payment/success`,
  failUrl: `${window.location.origin}/payment/fail`,
});
```

### 3. 결제 결과 페이지

- `/payment/success` — 토스에서 리다이렉트 → 백엔드 승인 API 호출 → 성공 화면
- `/payment/fail` — 실패 사유 표시 + 재시도 버튼

### 4. 라우트 추가

```
/payment/:orderId    → 결제 페이지 (위젯)
/payment/success     → 결제 성공 처리
/payment/fail        → 결제 실패 처리
```

## 백엔드 작업 항목

### 1. 토스페이먼츠 Secret Key 설정

```yaml
# application.yml
tosspayments:
  secret-key: test_sk_xxxxxxxxxxxxx
  client-key: test_ck_xxxxxxxxxxxxx
```

### 2. 결제 승인 API

```
POST /api/payments/confirm
```

요청:
```json
{
  "paymentKey": "토스에서 받은 키",
  "orderId": "BK-20260410-xxx",
  "amount": 10800
}
```

처리 순서:
1. `orderId`로 주문 조회
2. `amount`와 주문 `totalAmount` 비교 (위변조 검증)
3. 토스 승인 API 호출: `POST https://api.tosspayments.com/v1/payments/confirm`
4. 승인 성공 시 `payments` 테이블 저장
5. 주문 상태 `PENDING_PAYMENT -> PAID` 변경
6. 결과 반환

### 3. payments 테이블

```sql
CREATE TABLE payments (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    order_id BIGINT NOT NULL REFERENCES orders(id),
    payment_key VARCHAR(200) NOT NULL UNIQUE,
    method VARCHAR(30),              -- 카드, 계좌이체, 간편결제 등
    total_amount INTEGER NOT NULL,
    status VARCHAR(30) NOT NULL,     -- DONE, CANCELLED, ABORTED 등
    requested_at TIMESTAMP,
    approved_at TIMESTAMP,
    receipt_url VARCHAR(500),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4. Payment 엔티티

```java
@Entity
@Table(name = "payments")
public class Payment extends BaseEntity {
    @OneToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "order_id", nullable = false)
    private Order order;

    @Column(nullable = false, unique = true, length = 200)
    private String paymentKey;

    @Column(length = 30)
    private String method;

    @Column(nullable = false)
    private Integer totalAmount;

    @Column(nullable = false, length = 30)
    private String status;

    private LocalDateTime requestedAt;
    private LocalDateTime approvedAt;

    @Column(length = 500)
    private String receiptUrl;
}
```

### 5. 토스 승인 API 호출 (RestClient)

```java
RestClient restClient = RestClient.builder()
    .baseUrl("https://api.tosspayments.com/v1")
    .defaultHeader("Authorization", "Basic " + Base64.encode(secretKey + ":"))
    .build();

Map<String, Object> body = Map.of(
    "paymentKey", paymentKey,
    "orderId", orderId,
    "amount", amount
);

var response = restClient.post()
    .uri("/payments/confirm")
    .body(body)
    .retrieve()
    .body(Map.class);
```

### 6. OrderStatus 확장

```java
public enum OrderStatus {
    PENDING_PAYMENT,    // 주문 생성, 결제 대기
    PAID,               // 결제 완료
    PAYMENT_FAILED,     // 결제 실패
    CANCELLED           // 취소
}
```

| 순서 | 작업 | 영역 |

## 3차 완료 기준

아래가 되면 3차 구현 완료로 본다.

1. 주문 생성 후 토스 결제 위젯이 표시된다.
2. 테스트 카드로 결제가 완료된다.
3. 백엔드에서 결제 승인을 검증하고 payments에 저장한다.
4. 주문 상태가 `PAID`로 변경된다.
5. 결제 실패 시 에러 메시지가 표시되고 재시도할 수 있다.
6. 주문 상세에서 결제 완료 여부를 확인할 수 있다.
