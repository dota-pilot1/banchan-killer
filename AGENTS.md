# AGENTS.md

이 저장소에서 작업하는 에이전트는 아래 규칙을 우선적으로 따른다.

## 프로젝트 컨텍스트

- 이 서비스는 1인 가구를 위한 반찬 전문 쇼핑몰이다.
- 서비스 톤은 프리미엄 반찬몰보다 실속형, 자취형, 일상형에 가깝다.
- 상품명과 카피는 비싸 보이는 레스토랑식 표현보다 익숙한 반찬 이름을 우선한다.
- 예시: `꽈리고추 볶음`, `메추리알 볶음`, `파김치`, `콩나물 무침`

## DB 작업 규칙

- `spring.jpa.hibernate.ddl-auto`는 항상 `update`를 유지한다.
- 명시적 요청 없이 `create`, `create-drop`, 테이블 전체 삭제, 강제 초기화를 하지 않는다.
- 사용자나 운영성 데이터로 볼 수 있는 기존 데이터는 임의로 삭제하거나 덮어쓰지 않는다.
- 샘플 데이터는 로컬 개발 편의를 위한 목적일 때만 넣는다.
- 샘플 데이터는 테이블이 비어 있을 때만 삽입하는 것을 기본으로 한다.
- 기존 샘플 데이터를 교체해야 하면, 명확히 식별 가능한 legacy 샘플 데이터일 때만 교체한다.
- 데이터 마이그레이션이 필요한 구조 변경이면 영향 범위를 먼저 확인하고 무작정 엔티티만 바꾸지 않는다.

## 상품/카테고리 규칙

- 메인 헤더 기준 카테고리는 `야채 반찬`, `고기 반찬`, `김치류` 3개를 사용한다.
- 프론트 카테고리 페이지는 `/category/vegetable`, `/category/meat`, `/category/kimchi`를 기준으로 유지한다.
- 백엔드 enum은 프론트 카테고리보다 넓을 수 있지만, 프론트 매핑은 항상 명확해야 한다.
- 카테고리명을 바꾸거나 추가할 때는 백엔드 enum, 프론트 타입, 라우트, 문구를 함께 확인한다.

## UI/카피 규칙

- 홈 화면은 랜딩 페이지 역할을 유지하고, 카테고리 탐색은 별도 페이지 이동 방식으로 처리한다.
- 문구는 "혼자 살아도", "한 끼", "소포장", "부담 없는 가격", "자취 식탁" 같은 맥락을 우선한다.
- "프리미엄", "명품", "국가대표 맛집"처럼 과한 표현은 기본값으로 사용하지 않는다.
- 사용자가 바로 이해할 수 있는 짧은 카피를 우선한다.

## 작업 방식

- 프론트 변경 시 가능하면 타입 체크 또는 빌드 가능한 범위까지 검증한다.
- 백엔드 변경 시 최소한 컴파일이 깨지지 않는지 확인한다.
- 기존에 사용자가 만든 변경사항이 있으면 되돌리지 않는다.
- 규칙과 구현이 충돌하면 먼저 현재 코드와 데이터를 확인한 뒤 최소 변경으로 맞춘다.
