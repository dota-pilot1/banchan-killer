# 상품 기능 구현 계획 (Product CRUD)

## 1. 개요
반찬 킬러 쇼핑몰의 핵심 도메인인 '상품(Product)'에 대한 관리 및 조회 기능을 구현합니다.
초기 단계에서는 상품 등록(C)과 전체/단건 조회(R)를 동시에 진행하여 데이터 흐름을 완성합니다.

## 2. 도메인 모델 설계 (Product Entity)
| 필드명 | 타입 | 설명 | 제약 조건 |
| :--- | :--- | :--- | :--- |
| id | Long | 고유 식별자 | PK, Auto Increment |
| name | String | 상품명 | Not Null, Max 100 |
| description | String | 상품 상세 설명 | Not Null |
| price | Integer | 판매가 | Not Null, Min 0 |
| stockQuantity | Integer | 재고 수량 | Not Null, Min 0 |
| category | String | 카테고리 (밑반찬, 국/찌개 등) | Not Null |
| imageUrl | String | 상품 썸네일 경로 | - |
| status | Enum | 판매 상태 (SALE, SOLDOUT, HIDDEN) | Default SALE |

## 3. 패키지 구성 (DDD)
* `com.banchan.product.domain`: `Product`, `ProductRepository` (인터페이스)
* `com.banchan.product.application`: `ProductService` (비즈니스 로직)
* `com.banchan.product.presentation`: `ProductController`, `ProductRequest`, `ProductResponse`
* `com.banchan.product.infrastructure`: `ProductJpaRepository`, `ProductRepositoryImpl`

## 4. API 스펙 설계
### 4-1. 상품 등록 (Admin 용)
*   **Method**: `POST`
*   **Path**: `/api/admin/products`
*   **Auth**: ADMIN 권한 필요 (현재는 인증 통과 시 모두 허용으로 우선 구현)

### 4-2. 상품 목록 조회 (Public)
*   **Method**: `GET`
*   **Path**: `/api/products`
*   **Filter**: 카테고리별 필터링 (추후 추가)

### 4-3. 상품 상세 조회 (Public)
*   **Method**: `GET`
*   **Path**: `/api/products/{id}`

## 5. 구현 순서
1.  **Backend**: `Product` 엔티티 및 Repository 구현
2.  **Backend**: `ProductService`에서 등록 및 조회 로직 구현
3.  **Backend**: `ProductController` API 엔드포인트 개설
4.  **Frontend**: 상품 타입 정의 및 API 연동 (React Query)
5.  **Frontend**: 메인 페이지에 상품 목록 렌더링
