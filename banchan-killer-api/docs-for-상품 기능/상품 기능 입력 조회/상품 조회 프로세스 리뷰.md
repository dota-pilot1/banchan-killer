# 상품 조회 주요 코드 흐름 (Listing & Detail) - 고도화 버전

## 패키지 구조 (Standard Product Architecture)
```
com.banchan
├── product/
│   ├── domain/          ← Product (Entity), ProductCategory (Enum), ProductNotFoundException (Custom)
│   ├── application/     ← ProductService (Pagination 지원)
│   ├── presentation/    ← ProductController (Pageable 지원)
│   └── infrastructure/  ← ProductJpaRepository, ProductRepositoryImpl
└── common/
    └── exception/       ← GlobalExceptionHandler (404 Not Found 처리)
```

## Step 1. 클라이언트 요청 (Pagination & Enum 필터)

### 1-1. 상품 목록 페이징 조회
```http
GET /api/products?category=SIDE_DISH&page=0&size=20&sort=id,desc
Accept: application/json
```

## Step 2. ProductController (Presentation Layer)
`Pageable`과 `ProductCategory` Enum을 직접 파라미터로 받아 타입 안정성을 확보합니다.
```java
@GetMapping
public ResponseEntity<Page<ProductResponse>> getProducts(
        @PageableDefault(size = 20) Pageable pageable,
        @RequestParam(required = false) ProductCategory category) {
    return ResponseEntity.ok(productService.getProducts(pageable, category));
}
```

## Step 3. ProductService (Application Layer)
데이터베이스 조회를 수행하고, `Page<Product>`를 `Page<ProductResponse>`로 변환하여 반환합니다.
```java
public Page<ProductResponse> getProducts(Pageable pageable, ProductCategory category) {
    if (category != null) {
        return productRepository.findByCategory(category, pageable)
                .map(ProductResponse::from);
    }
    return productRepository.findAll(pageable)
            .map(ProductResponse::from);
}
```

## Step 4. ProductNotFoundException (Domain Layer)
상품이 존재하지 않을 때 도메인 언어로 명확하게 예외를 던집니다.
```java
Product product = productRepository.findById(id)
    .orElseThrow(() -> new ProductNotFoundException(id));
```

## Step 5. 응답 결과 (Page 객체 포함)
성공 시 단순 목록이 아닌 **페이징 정보(totalElements, totalPages 등)**가 포함된 응답을 받습니다.
```json
{
  "content": [
    {
      "id": 1,
      "name": "매콤 멸치볶음",
      "category": "SIDE_DISH",
      "price": 4500,
      "status": "SALE"
      ...
    }
  ],
  "pageable": { ... },
  "totalPages": 5,
  "totalElements": 100
}
```

---

## 🛡️ 핵심 고도화 포인트
*   **Type Safety**: 카테고리를 단순 문자열이 아닌 `ProductCategory` Enum으로 관리하여 오타를 방지합니다.
*   **Error Handling**: `GlobalExceptionHandler`와 전용 예외를 연결해 404 에러를 명확히 처리합니다.
*   **Performance**: `Pagination`을 통해 대량의 상품 데이터도 효율적으로 처리할 수 있습니다.
