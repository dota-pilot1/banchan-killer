# 로그인 주요 코드 흐름 (Email + JWT 기반)

## 패키지 구조 (Standard JWT Architecture)
```
com.banchan
├── auth/
│   ├── presentation/    ← AuthController, LoginRequest, LoginResponse
│   ├── application/     ← AuthService
│   └── domain/          ← InvalidCredentialsException
├── user/
│   ├── domain/          ← User, UserRepository (순수 인터페이스)
│   └── infrastructure/  ← UserJpaRepository, UserRepositoryImpl
├── config/
│   ├── jwt/             ← JwtTokenProvider, JwtAuthenticationFilter
│   └── SecurityConfig   ← SecurityFilterChain Stateless 설정
└── common/
    └── exception/       ← GlobalExceptionHandler
```

## Step 1. 클라이언트 요청 (Email 기반)
```http
POST /api/auth/login
Content-Type: application/json

{
  "email": "terecal@daum.net",
  "password": "hyun0316!@"
}
```

## Step 2. AuthController (Presentation Layer)
`@Valid`를 통해 이메일 형식 및 필수값 검증 후, 서비스 계층으로 처리를 위임합니다.
```java
@PostMapping("/login")
public ResponseEntity<LoginResponse> login(@Valid @RequestBody LoginRequest request) {
    LoginResponse response = authService.login(request);
    return ResponseEntity.ok(response);
}
```

## Step 3. AuthService (Application Layer) - 비즈니스 로직
이제 아이디(`username`)가 아닌 **이메일**을 식별자로 사용합니다.

### 3-1. 회원 조회 및 비밀번호 대조
```java
User user = userRepository.findByEmail(request.email())
    .orElseThrow(InvalidCredentialsException::new);

if (!passwordEncoder.matches(request.password(), user.getPassword())) {
    throw new InvalidCredentialsException();
}
```

### 3-2. JWT 토큰 생성
`JwtTokenProvider`를 호출하여 유저 정보가 담긴 Access Token을 생성합니다.
```java
String token = jwtTokenProvider.createToken(user.getEmail(), user.getRole().name());
```

## Step 4. JwtTokenProvider (Infrastructure/Config Layer)
`jjwt` 라이브러리를 사용해 토큰을 암호화하고 서명합니다.
* **Claims**: 이메일(Subject), 권한(Role) 포함
* **Sign**: HS256 알고리즘 사용

## Step 5. 응답 결과
성공 시 프론트엔드는 이 토큰을 받아 `localStorage`에 저장하고, 이후 모든 요청 헤더에 `Authorization: Bearer <Token>`을 실어 보냅니다.
```json
{
  "id": 1,
  "email": "user@example.com",
  "nickname": "반찬킬러",
  "role": "USER",
  "accessToken": "eyJhbGciOiJIUzI1NiJ..."
}
```

---

## 🛡️ 인증(Authorization) 프로세스 (이후 요청 시)
로그인 이후의 요청(장바구니 담기 등)은 `JwtAuthenticationFilter`가 가로챕니다.
1. `resolveToken()`: 헤더에서 Bearer 토큰 추출
2. `validateToken()`: 위변조 및 만료 여부 검사
3. `SecurityContextHolder`: 인증이 성공하면 시큐리티 컨텍스트에 유저를 등록하여 컨트롤러에서 `@AuthenticationPrincipal` 등으로 쓸 수 있게 합니다.
