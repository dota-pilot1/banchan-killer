# 회원 가입 주요 코드 흐름 (ID 제거 및 UX 최적화)

## 설계 원칙 (UX + DDD)
* **ID 제거**: 기억하기 어려운 아이디(username)를 없애고 **이메일을 고유 식별자**로 사용해 가입 허들을 낮췄습니다.
* **Auto Login**: 가입 성공 즉시 로그인 처리하여 사용자가 바로 서비스를 이용할 수 있도록 UX를 설계했습니다.

## Step 1. 클라이언트 요청 (SignupRequest)
```http
POST /api/auth/signup
Content-Type: application/json

{
  "email": "newuser@example.com",
  "password": "password123",
  "nickname": "반찬러버"
}
```

## Step 2. AuthService (Application Layer) - 비즈니스 로직

### 2-1. 이메일 중복 검사
도메인 인터페이스 `UserRepository.existsByEmail`을 사용하여 중복 가입을 방지합니다. 중복 시 `DuplicateEmailException`(409 Conflict)이 발생합니다.

### 2-2. 비밀번호 암호화 및 도메인 모델 생성
`BCryptPasswordEncoder`로 해싱한 비밀번호를 가진 `User` 도메인 엔티티를 생성합니다. 이때 `id`는 DB 저장 시점에 생성됩니다.
```java
User user = new User(
    request.email(),
    passwordEncoder.encode(request.password())
);
```

### 2-3. 영속화 (Persistence)
인프라 계층(JPA)에 저장을 위임하여 `users` 테이블에 새로운 행을 추가합니다.

## Step 3. 응답 결과 (SignupResponse)
가입된 유저의 정보를 반환합니다. 이전과 달리 응답 결과에 `username` 필드가 존재하지 않습니다.
```json
{
  "id": 2,
  "email": "newuser@example.com",
  "nickname": "반찬러버",
  "role": "USER"
}
```

## Step 4. 프론트엔드 자동 로그인 (Feature Layer)
가입 통신(`POST /signup`)이 성공하면, 프론트엔드(`SignupForm.tsx`)는 내부적으로 즉시 로그인 통신을 시도합니다.
1. `apiClient.post('/auth/signup', ...)` 성공
2. `apiClient.post('/auth/login', { email: 'terecal@daum.net', password: 'hyun0316!@' })` 바로 실행
3. 반환된 JWT 토큰을 저장하고 홈 화면으로 이동

---

## 🏗️ DB 스키마 점검 사항
* **users 테이블**: `username` 컬럼이 완전히 삭제되었습니다.
* **제약 조건**: `email` 컬럼에 `UNIQUE`, `NOT NULL` 인덱스가 적용되어 데이터 정합성을 보장합니다.
