# S3 이미지 업로드 프로세스 정리

이 문서는 관리자 상품 화면에서 이미지를 업로드할 때, 프론트엔드부터 백엔드 S3 저장까지 어떤 순서로 동작하는지 정리한 문서입니다.

## Step 1. 관리자 화면에서 이미지 파일을 선택한다

- 파일 경로
  - [`banchan-killer-front/src/pages/admin/ui/AdminProductsPage.tsx`](/Users/terecal/banchan-killer-container/banchan-killer-front/src/pages/admin/ui/AdminProductsPage.tsx)
- 내용
  - 오른쪽 상품 수정 패널에서 카메라 버튼 또는 `이미지 변경` 버튼을 누르면 숨겨진 `input type="file"`이 열립니다.
  - 사용자가 이미지를 선택하면 `handleImageUpload()`가 실행됩니다.
  - 이 함수는 선택한 파일을 `FormData`에 `file` 키로 담아 `/uploads/images`로 전송합니다.

## Step 2. 프론트엔드가 업로드 API를 호출한다

- 파일 경로
  - [`banchan-killer-front/src/pages/admin/ui/AdminProductsPage.tsx`](/Users/terecal/banchan-killer-container/banchan-killer-front/src/pages/admin/ui/AdminProductsPage.tsx)
  - [`banchan-killer-front/src/shared/api/base.ts`](/Users/terecal/banchan-killer-container/banchan-killer-front/src/shared/api/base.ts)
- 내용
  - `apiClient.post('/uploads/images', formData, { headers: { 'Content-Type': 'multipart/form-data' } })` 요청이 실행됩니다.
  - `apiClient`의 기본 `baseURL`이 `/api`이므로 실제 요청 경로는 `/api/uploads/images`가 됩니다.
  - 업로드 성공 시 응답에서 받은 `imageUrl` 값을 현재 상품 수정 폼 상태에 넣습니다.
  - 업로드 실패 시 백엔드에서 내려준 메시지를 `error` 상태로 보여줍니다.

## Step 3. Spring Boot 업로드 컨트롤러가 파일을 받는다

- 파일 경로
  - [`banchan-killer-api/src/main/java/com/banchan/upload/presentation/ImageUploadController.java`](/Users/terecal/banchan-killer-container/banchan-killer-api/src/main/java/com/banchan/upload/presentation/ImageUploadController.java)
- 내용
  - `POST /api/uploads/images` 요청은 `ImageUploadController.uploadImage()`로 들어옵니다.
  - 컨트롤러는 `@RequestParam("file") MultipartFile file` 로 파일을 받습니다.
  - 받은 파일을 `ImageUploadService.uploadProductImage(file)`로 넘깁니다.

## Step 4. S3 업로드 서비스가 파일 유효성을 검사한다

- 파일 경로
  - [`banchan-killer-api/src/main/java/com/banchan/upload/application/ImageUploadService.java`](/Users/terecal/banchan-killer-container/banchan-killer-api/src/main/java/com/banchan/upload/application/ImageUploadService.java)
- 내용
  - `validateFile()`에서 먼저 업로드 가능 여부를 확인합니다.
  - 확인 항목은 아래와 같습니다.
    - `bucketName`이 비어 있는지
    - 파일이 비어 있는지
    - 파일 크기가 제한을 넘는지
    - `image/jpeg`, `image/png`, `image/webp`, `image/gif` 형식인지
  - 조건을 만족하지 않으면 `FileUploadException`을 발생시켜 업로드를 중단합니다.

## Step 5. S3 객체 키를 만든다

- 파일 경로
  - [`banchan-killer-api/src/main/java/com/banchan/upload/application/ImageUploadService.java`](/Users/terecal/banchan-killer-container/banchan-killer-api/src/main/java/com/banchan/upload/application/ImageUploadService.java)
- 내용
  - `buildObjectKey()`가 업로드할 파일 경로를 만듭니다.
  - 저장 키 형식은 아래와 같습니다.
  - `products/YYYY/MM/DD/UUID.확장자`
  - 예를 들면 `products/2026/04/10/uuid-value.jpg` 같은 형태입니다.
  - 이렇게 날짜별 폴더 구조로 저장되므로 운영 중 파일을 관리하기가 편합니다.

## Step 6. S3Client가 실제 파일을 버킷에 업로드한다

- 파일 경로
  - [`banchan-killer-api/src/main/java/com/banchan/upload/application/ImageUploadService.java`](/Users/terecal/banchan-killer-container/banchan-killer-api/src/main/java/com/banchan/upload/application/ImageUploadService.java)
  - [`banchan-killer-api/src/main/java/com/banchan/config/S3Config.java`](/Users/terecal/banchan-killer-container/banchan-killer-api/src/main/java/com/banchan/config/S3Config.java)
  - [`banchan-killer-api/src/main/resources/application.properties`](/Users/terecal/banchan-killer-container/banchan-killer-api/src/main/resources/application.properties)
- 내용
  - `S3Config`에서 `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `aws.s3.region` 값을 사용해 `S3Client` Bean을 생성합니다.
  - `ImageUploadService`는 `PutObjectRequest`를 만들어 `bucket`, `key`, `contentType`을 설정합니다.
  - 이후 `s3Client.putObject()`로 실제 파일 바이트를 S3에 업로드합니다.
  - 업로드 성공 시 서비스는 최종 공개 URL 문자열을 만들어 반환합니다.

## Step 7. 백엔드가 이미지 URL을 프론트엔드에 반환한다

- 파일 경로
  - [`banchan-killer-api/src/main/java/com/banchan/upload/presentation/ImageUploadController.java`](/Users/terecal/banchan-killer-container/banchan-killer-api/src/main/java/com/banchan/upload/presentation/ImageUploadController.java)
  - [`banchan-killer-api/src/main/java/com/banchan/upload/presentation/ImageUploadResponse.java`](/Users/terecal/banchan-killer-container/banchan-killer-api/src/main/java/com/banchan/upload/presentation/ImageUploadResponse.java)
- 내용
  - 컨트롤러는 서비스에서 받은 이미지 URL을 `ImageUploadResponse`에 담아 JSON으로 응답합니다.
  - 프론트엔드는 이 응답의 `imageUrl`을 폼에 반영하고, 오른쪽 미리보기 이미지에도 바로 보여줍니다.

## Step 8. 최종 상품 수정 요청에서 imageUrl을 저장한다

- 파일 경로
  - [`banchan-killer-front/src/pages/admin/ui/AdminProductsPage.tsx`](/Users/terecal/banchan-killer-container/banchan-killer-front/src/pages/admin/ui/AdminProductsPage.tsx)
- 내용
  - 이미지 업로드만으로 상품 DB가 바로 수정되지는 않습니다.
  - 업로드 결과 URL은 먼저 프론트 폼 상태에만 들어갑니다.
  - 사용자가 마지막에 `상품 수정하기` 또는 상품 등록 버튼을 눌러야, 해당 `imageUrl`이 상품 수정/등록 API와 함께 저장됩니다.

## Step 9. 예외 상황은 공통 예외 처리기에서 메시지로 내려준다

- 파일 경로
  - [`banchan-killer-api/src/main/java/com/banchan/common/exception/GlobalExceptionHandler.java`](/Users/terecal/banchan-killer-container/banchan-killer-api/src/main/java/com/banchan/common/exception/GlobalExceptionHandler.java)
  - [`banchan-killer-api/src/main/java/com/banchan/common/exception/FileUploadException.java`](/Users/terecal/banchan-killer-container/banchan-killer-api/src/main/java/com/banchan/common/exception/FileUploadException.java)
- 내용
  - S3 버킷명이 없거나, 확장자가 맞지 않거나, 업로드 중 예외가 나면 `FileUploadException`이 발생합니다.
  - 파일 크기가 너무 크면 `MaxUploadSizeExceededException`도 공통 예외 처리기에서 잡습니다.
  - 예외 메시지는 프론트에서 그대로 받아 관리자 화면 에러 문구로 보여줍니다.

## Step 10. 현재 설정상 반드시 맞춰야 하는 환경변수

- 파일 경로
  - [`banchan-killer-api/.env`](/Users/terecal/banchan-killer-container/banchan-killer-api/.env)
  - [`banchan-killer-api/src/main/resources/application.properties`](/Users/terecal/banchan-killer-container/banchan-killer-api/src/main/resources/application.properties)
- 내용
  - 현재 S3 업로드에 필요한 핵심 값은 아래와 같습니다.
  - `AWS_ACCESS_KEY_ID`
  - `AWS_SECRET_ACCESS_KEY`
  - `AWS_S3_BUCKET_NAME` 또는 `S3_BUCKET_NAME`
  - `AWS_S3_REGION` 또는 `AWS_REGION`
  - 이 값이 비어 있으면 업로드 서비스에서 바로 예외를 발생시킵니다.

## 참고

- 현재 Spring multipart 최대 업로드 제한은 `10MB`입니다.
- 다만 `ImageUploadService` 내부 검증은 현재 `5MB` 기준을 사용하고 있습니다.
- 따라서 실제 운영 기준으로는 더 엄격한 `5MB 이하`만 업로드 가능하다고 이해하면 됩니다.
